Strategy

A PIT Crew should never start from a blank page

On July 13, 2026 New York launched the Public Interest Technology (PIT) Crew to “rapidly build digital solutions to public problems” — in months, not years. This project is the substrate that makes that speed real: for all 67 agencies the models, contracts, agent tools, and the missing write workflows are already drafted, open, and forkable.

The through-line

A fifteen-year argument, now proven

API Evangelist has made this case since 2011. The NYC assessment is the evidence — the open-data era (2010–2018) half-worked; it published reporting data and never touched the transaction.

The opportunity

How New York leads other cities

The open-data era had a standard — but it was a vendor's (Socrata/SODA). This time the standard is open, forkable, and agent-native, and New York authors it.

Standards

nyc-commons geography/identity spine · an APIs.json catalog · the ten government-process skills · the design-first chain.

Tooling

The experience layer · a callable reference API · the installable @api-common/nyc-mcp · the open generators.

Processes

The seven-step method and the whole repository — forkable as a civic API kit. Any city clones it and runs its own assessment.

The ask

What to do next

  1. Fork the DCWP complaint contract as crew #1's starting point.
  2. Adopt nyc-commons as the citywide geography/identity spine.
  3. Register every agency in an APIs.json catalog — the connective tissue open data never had.
  4. Make an MCP surface a standard deliverable for every PIT Crew build.
  5. Require an open-source-alternative assessment for every new proprietary purchase.
  6. Publish the kit so other cities fork it — NYC as the reference city.
